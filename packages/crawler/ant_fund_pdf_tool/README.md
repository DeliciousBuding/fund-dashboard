# Ant/Alipay 基金交易明细 PDF 转换器

适配对象：蚂蚁（杭州）基金销售有限公司导出的“基金（含资产管理计划）交易明细”PDF，表头包含：

`订单号 / 交易时间 / 交易类型 / 基金名称 / 组合基金名称 / 基金代码 / 申请金额 / 申请份额 / 确认金额 / 确认份额 / 手续费 / 确认日期`

## 解决的问题

1. **页首续行合并**：PDF 表格跨页时，上一页最后一条交易可能被拆到下一页页首。脚本会把只有订单号尾段、基金名称尾段的行合并回上一条交易。
2. **保留代码格式**：基金代码保持 6 位字符串，例如 `000218`、`008971`，不会变成 `218`、`8971`。
3. **日期断行修复**：把 `2026/06/0\n9 11:41` 解析为 `2026-06-09 11:41`。
4. **转换语义修复**：
   - `用户跨TA转换`：同一订单通常有转入/转出两行，按金额/份额判断方向。
   - `用户转换`：PDF 单行通常只显示目标基金。脚本把该行作为目标基金转入，再根据转换前持仓余额和最终残差自动补一条 synthetic 源基金转出。
5. **QA 输出**：输出续行日志、转换推断日志、异常复核表，避免静默错账。

## 使用

```bash
python ant_fund_pdf_converter.py input.pdf --out export
```

依赖：

```bash
pip install pymupdf
```

## 输出文件

- `transactions_raw_merged.csv`：按 PDF 网格抽出的原始交易行，已合并页首续行。
- `transactions_normalized.csv`：清洗后的完整流水，包含 synthetic 的 `用户转换` 源基金转出行。
- `summary_by_fund_corrected.csv`：按基金汇总。
- `current_positions_corrected.csv`：按交易流水反推的非零净份额/当前持仓候选。
- `normal_conversion_review.csv`：`用户转换` 的源/目标基金推断过程。
- `manual_review_issues.csv`：需要人工复核的问题。若只有货币基金收益差额，一般不影响普通基金持仓判断。
- `qa_report_corrected.json`：总 QA 报告。
- `continuation_merge_log.json`：页首续行合并日志。

## 注意

PDF 官方注释说明：交易明细仅反映查询日基金交易账号记录的已确认份额，不含在途交易；若与注册登记机构记录不一致，以注册登记机构为准。因此这里导出的“当前持仓”是根据交易流水反推的净份额，不等价于 App 官方实时持仓快照。
